# AGENTS.md

## 言語
- コードコメント、README、設計/運用ドキュメント、計画書は日本語で書く。

## 変更方針
- 変更は最小限にする。
- 無関係な整形、不要なリネーム、不要なファイル移動はしない。

## 依頼の種類ごとの動作

### 提案・質問（「どう思う？」「案を出して」など）
- 既存ファイルは書き換えず、回答のみ返す。
- ただし、内容確認を目的としたコマンド実行は行ってよい。
- ただし、内容を伝えるための文書作成（新規ファイル）は必要な場合に限り行ってよい（既存ファイルの編集はしない）。

### 計画（「計画して」と要求された場合）
- `./plan/` 配下に計画書を1つ作成する（ディレクトリがなければ作成する）。
- 計画書のファイル名は `YYYY-MM-DD-<title>.md` を基本とする。
- 計画書には少なくとも以下を含める：
  - 目的 / スコープ
  - 具体手順（順序つき）
  - 影響範囲・リスク
  - 検証観点（コマンドは後で追記でもよい）
- 計画の提示後、明示的に「計画を実行して」と要求されるまで実装は開始しない。

### 計画の実行（「計画を実行して」と要求された場合）
- 原則として「直近に作成/更新した計画書」を実行対象にし、その内容に沿って作業する。
- 計画と異なる対応が必要になった場合は、その理由と変更点を記録しながら進める。

## 完了時の作業
- 計画が完了したら、計画書に以下を追記する：
  - 実施結果（実際に行ったこと）
  - 計画との差分（計画と異なる箇所、理由、判断）
- 計画が完了したら、関連するREADME（必要に応じて root README / 該当ディレクトリ配下のREADME）を適切に修正する。
